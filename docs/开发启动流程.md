# WC Predictor 开发启动流程

本文档说明本地如何启动前后端、各运行模式的区别，以及常见问题排查。

---

## 1. 架构速览

```
浏览器 (localhost:3000)
    │
    │  npm run dev (Vite)
    ▼
frontend/  ── /api/* 代理 ──►  Go 后端 (localhost:8080)
                                    │
                                    ├── mock/          ← DATA_SOURCE=mock
                                    └── GitHub 等外网   ← DATA_SOURCE=live
```

- **前端**：React + Vite，目录 `frontend/`，开发端口 **3000**
- **后端**：Go + Cobra + HTTP Server，项目根目录，端口 **8080**
- **联调方式**：前端通过 Vite 代理把 `/api/*` 转发到后端（见 `frontend/vite.config.ts`）

| 工具 | 用途 | 检查命令 |


## 3. 推荐：完整联调（前后端各开一个终端）

### 终端 1 — 启动 Go 后端

```powershell
cd E:\自己\工作\Wc-System

# 开发推荐：使用本地 mock 数据，不依赖外网
$env:DATA_SOURCE = "mock"

go run . serve
```

看到以下日志表示成功：

```
[cache] refresh complete: 24 teams loaded
[server] listening on 0.0.0.0:8080
```

验证：

```powershell
Invoke-WebRequest -Uri "http://localhost:8080/api/health" -UseBasicParsing
# 期望返回: {"status":"ok","last_refresh":"..."}
```

### 终端 2 — 启动 React 前端

```powershell
cd E:\自己\工作\Wc-System\frontend
npm run dev
```

浏览器访问：**http://localhost:3000**

### 联调时的行为

- 页面顶部显示 **「Go 引擎: 在线」** → 预测走 Go API
- 点击 **「运行矩阵运算」** → `POST /api/predict` → Go 引擎计算并返回结果
- 修改参数时，界面会先用 **前端本地算法** 做即时预览；正式运算以按钮触发后端为准

## 4. 三种运行模式对比

### 4.1 完整联调（前后端都开）

| 项目 | 说明 |
|------|------|
| 适用 | 日常开发、验证 API、联调 UI |
| 启动 | `go run . serve` + `npm run dev` |
| 计算引擎 | Go 后端（与 PRD 一致） |
| 数据来源 | 由 `DATA_SOURCE` 决定（见 4.3） |

### 4.2 仅前端（后端未启动）

| 项目 | 说明 |
|------|------|
| 适用 | 只改 UI、样式、交互 |
| 启动 | 仅 `npm run dev` |
| 计算引擎 | 浏览器内 TypeScript（`frontend/src/utils/predictor.ts`） |
| 状态 | 顶部显示 **「Go 引擎: 离线 (本地模式)」** |
| 点击运算 | 尝试调 API 失败后自动降级，并显示黄色警告横幅 |

> 本地降级算法与 Go 引擎公式不完全相同，**验收预测逻辑请以 Go 后端为准**。

### 4.3 Docker 仅后端

| 项目 | 说明 |
|------|------|
| 适用 | 部署/验证容器化后端 |
| 启动 | 项目根目录 `docker compose up --build` |
| 前端 | **不包含**在 compose 中，仍需单独 `npm run dev` |
| 端口 | 后端映射 `8080:8080` |

## 5. 「离线开发」是什么意思？

**离线** 指的是 **数据获取方式**，不是「整个系统断网可用」。

| 模式 | 环境变量 | 数据来源 | 是否需要外网 |
|------|----------|----------|--------------|
| Mock / 离线 | `DATA_SOURCE=mock` | `mock/results.csv`、`mock/elo_ratings.csv` | 否 |
| Live / 在线 | `DATA_SOURCE=live`（默认） | GitHub 赛果 CSV、eloratings.net Elo、World Bank 等 | 是 |

Live 模式下 Elo 从 [eloratings.net/World.tsv](https://www.eloratings.net/World.tsv) 拉取（通过 `data/wc2026_teams.json` 中的 `elo_code` 映射）。若拉取失败，会自动降级到 `mock/elo_ratings.csv`，服务仍可启动。

### Mock 模式适合

- 无外网或外网不稳定
- 快速验证引擎、CLI、API
- CI / 本地单元测试

### Mock 数据位置

```
mock/
├── results.csv      # 约 160 行国际比赛记录
└── elo_ratings.csv  # 48 支世界杯球队 Elo（离线备用）
```

### 仍需网络的功能

| 功能 | 条件 |
|------|------|
| DeepSeek AI 情报修正 | 需配置 `DEEPSEEK_API_KEY` 且 `use_deepseek=true` |
| Live 数据源刷新 | `DATA_SOURCE=live` 或 `predictor fetch` |

## 6. CLI 快速验证（不启 HTTP）

适合只测 Go 计算链路：

```powershell
cd E:\自己\工作\Wc-System
$env:DATA_SOURCE = "mock"

# 基本预测
go run . calc --home ARG --away FRA

# 带主观干预
go run . calc --home Argentina --away France --home-mod -0.15 --avg-goals 2.3

# 手动指定 Elo
go run . calc --home BRA --away ENG --home-elo 2050 --away-elo 1980
```

手动刷新数据缓存：

```powershell
go run . fetch --source all    # 或 csv / elo / worldbank / wikimedia / fm
```

## 7. 环境变量速查

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DATA_SOURCE` | `live` | `mock` = 本地 CSV；`live` = 拉取外网 |
| `PORT` | `8080` | HTTP 服务端口 |
| `DEEPSEEK_API_KEY` | （空） | 配置后 UI 可开启 DeepSeek 修正 |
| `ELO_SLOPE` | `0.0011` | Elo→进球线性系数（0.0009–0.0014） |
| `FM_CSV_PATH` | （空） | Football Manager 导出 CSV 路径 |

PowerShell 临时设置示例：

```powershell
$env:DATA_SOURCE = "mock"
$env:PORT = "8080"
$env:DEEPSEEK_API_KEY = "sk-xxx"   # 可选
go run . serve
```

Docker 使用 `.env` 文件（从 `.env.example` 复制）。

## 9. 常见问题

### 端口 8080 已被占用

```powershell
# 查看占用进程（PowerShell）
Get-NetTCPConnection -LocalPort 8080 | Select-Object OwningProcess

# 或换端口启动
$env:PORT = "8081"
go run . serve --port 8081
```

若改后端端口，需同步修改 `frontend/vite.config.ts` 中 proxy 的 `target`。

### 前端显示「Go 引擎: 离线」

1. 确认后端已启动且 `/api/health` 可访问
2. 确认前端 `npm run dev` 已启动（代理仅在 dev 模式生效）
3. 检查防火墙是否拦截 localhost

### `team not found in data store`

- Mock 模式下球队名需能在 `mock/results.csv` / `mock/elo_ratings.csv` 中匹配
- 请求体可用 **球队 ID**（如 `ARG`）或 **英文名**（如 `Argentina`）

### modifier 返回 400

`home_mod` / `away_mod` 必须在 **[-0.5, +0.5]** 范围内。

## 10. 日常开发建议流程

```
1. 打开终端 A → DATA_SOURCE=mock → go run . serve
2. 打开终端 B → cd frontend → npm run dev
3. 浏览器打开 http://localhost:3000
4. 确认顶部「Go 引擎: 在线」
5. 改 Sidebar 参数 → 点击「运行矩阵运算」验证
6. 改 Go 代码后重启终端 A；改前端代码 Vite 热更新自动生效
```
