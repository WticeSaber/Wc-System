2. HTTP API 层新增：除原设计的 CLI 模式（predictor calc）外，系统额外实现了 predictor serve HTTP 服务器，为 React 前端提供 REST API。完整接口文档见 docs/API.md。

3. Elo 线性斜率参数化：原文档规定范围 0.0009-0.0014，代码默认值为 0.0011（范围中间值）。通过 ELO_SLOPE 环境变量可在不修改代码的前提下做敏感性测试。

4. DeepSeek 情报修正层（新增）：基于 OpenAI-compatible API，输入双队近期 5 场比赛结果，输出 [-0.2, +0.2] 范围内的动量修正系数。该系数在微操修正 (1+β) 之后、λ 软性断言之前叠加。API Key 通过 DEEPSEEK_API_KEY 环境变量配置，不配置时该层自动跳过（非阻断性）。

5. 附加数据适配器：World Bank GDP API、Wikimedia 球队介绍、Football Manager CSV 球员属性聚合均已实现为独立适配器，存储于 TeamRawStats.MacroContext 与 FMAttributes 字段，供未来权重融合扩展使用（当前不参与核心 λ 计算）。

6. 前端降级机制：当 Go 后端不可达时，前端自动回退至浏览器端 TypeScript Elo-泊松本地计算（predictor.ts 保留），确保离线可用性。用户界面以黄色提示横幅标注当前为"本地计算模式"。
